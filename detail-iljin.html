01 — 프로젝트 배경

분산된 웹사이트를 하나로 통합했습니다

회사 홈페이지와 개별 솔루션으로 분산되어 있던 3개의 웹사이트가 트래픽 분산과 정보 비일관성 문제를 일으키고 있었습니다. 고객 접근성을 높이고 마케팅·운영 효율을 개선하기 위해 하나의 도메인으로 통합하는 방향을 설정했습니다.

😩 문제
3개 사이트로
정보가 분산됐다
회사 홈페이지와 솔루션 사이트가 따로 운영되어 정보 일관성이 떨어지고 고객이 원하는 정보를 찾기 어려웠습니다.
✦ 해결
단일 도메인
통합 운영
기업 정보와 솔루션을 하나의 도메인으로 통합하여 고객이 자연스럽게 탐색하고 문의로 이어지는 흐름을 구성했습니다.
😩 문제
워드프레스로
유지보수가 어렵다
기존 홈페이지가 워드프레스로 개발되어 커스터마이징과 유지보수에 어려움이 있었고 업데이트가 거의 이루어지지 않았습니다.
✦ 해결
전체 개편 및
콘텐츠 최신화
워드프레스 방식에서 탈피하여 전체 개편을 진행하고 노후화된 비주얼과 콘텐츠를 전면 최신화했습니다.
😩 문제
고객 문의 채널이
흩어져 있다
각 사이트마다 문의 채널이 달라 고객이 혼란을 느끼고 담당자도 문의를 놓치는 상황이 발생했습니다.
✦ 해결
단일 창구로
문의 일원화
흩어져 있던 문의 채널을 하나로 통합하여 고객 접근성을 높이고 운영 효율을 개선했습니다.
02 — 프로젝트 목표

운영 도메인 통합 방향성을 정의했습니다

단순한 디자인 개선이 아닌, 운영 구조 자체를 바꾸는 작업이었습니다. 워드프레스 탈피, 콘텐츠 최신화, 인터랙션 개선, 고객 문의채널 통합, 디자인 개선을 5가지 핵심 목표로 설정하고 우선순위를 정립했습니다.

03 — 기업 홈페이지

기업 소개부터 고객사까지 하나의 흐름으로 구성했습니다

기존에는 회사와 솔루션 사이트가 따로 있어 고객이 정보를 파악하기 불편했습니다.

흩어져 있던 기업 정보와 솔루션을 하나로 모아 콘텐츠를 최신화했고, 회사의 신뢰가 제품 경쟁력으로 자연스럽게 이어지도록 페이지 흐름을 설계했습니다.

기업 정보 최신화
기업 소개, 제품, 기술력, 고객사를
하나의 흐름으로 재구성
신뢰→경쟁력 연결
회사 신뢰도가 솔루션 경쟁력으로
자연스럽게 이어지는 흐름 설계
인터랙션 개선
스크롤 Parallax·Scale 효과로
몰입감 있는 탐색 경험 제공
04 — 솔루션 랜딩 페이지

사용자의 행동 흐름을 예측해 구성했습니다

제품 개요·특장점으로 시작하여 기능 → 프로세스 → 요금제 → 도입문의로 이어지는 사용자의 자연스러운 탐색 흐름을 예측하고, 이탈 없이 문의로 연결되도록 설계했습니다.

고정형 서브 탭
스크롤 시 GNB 대신 서브 탭이 상단에 고정되어 제품 소개부터 문의 단계까지 탭 이동만으로 확인 가능하도록 했습니다.
몰입형 인터랙션
스크롤 위치에 반응하는 Parallax 및 Scale 효과를 적용하여 단순한 정보 습득을 몰입감 있는 경험으로 개선했습니다.
맞춤형 문의 설계
문의 유형을 구체화하고 불필요한 입력 항목을 줄였습니다. 필드 입력 시 완료 게이지를 표시해 사용성을 높였습니다.
05 — 화면 디자인

주요 화면을 설계했어요

기업 홈페이지부터 솔루션 랜딩 페이지까지 일관된 비주얼 시스템을 적용했습니다. 회사 정체성이 자연스럽게 드러나면서 방문자가 원하는 정보에 빠르게 도달하도록 페이지 구조와 인터랙션을 함께 설계했습니다.

06 — 회고

프로젝트 회고

단순히 디자인을 바꾸는 것이 아니라 운영 구조 자체를 재설계하는 프로젝트였습니다. 여러 사이트를 하나로 합치는 과정에서 정보 구조와 사용자 흐름에 대한 깊은 고민을 할 수 있었습니다.

통합의 관점
여러 사이트를 단순히 합치는 것이 아니라, 각 정보가 자연스럽게 연결되는 구조를 설계하는 것의 중요성을 배웠습니다.
사용자 행동 예측
방문자가 어떤 순서로 정보를 탐색하는지 예측하고, 그 흐름에 맞게 콘텐츠를 배치하는 역량을 키웠습니다.
문의 전환 설계
정보 탐색에서 문의로 이어지는 자연스러운 흐름을 만들기 위해 UX와 카피라이팅을 함께 고려하게 됐습니다.
